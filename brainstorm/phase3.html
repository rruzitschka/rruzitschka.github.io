Phase 3 Brainstorm

Phase 3 — What Are We Building?

Phase 2 (read-only logbook) is done. Phase 3 adds Training + richer Stats to the web dashboard.

Scope options — pick what goes in Phase 3
Option A · Lean

Training: Read-Only

Show training sessions from CloudKit, similar to the logbook tab.

  • Training tab active (no toast)
  • Fetch CD_TrainingSession records
  • List view: type, date, duration, intensity
  • Stats: total sessions, total time, avg/week
  • Filter by type / period
  • Bar chart: sessions by type
  • Add / edit / delete sessions
  • Combined climb+training stats
Option B · Full

Training: Read + Write

Full parity with the iOS Training tab — view and log sessions from the web.

  • Everything in Option A
  • Add new training session (form overlay)
  • Delete session (with confirmation)
  • Combined Stats tab: streak uses both climbs + training
  • Mock data for training sessions
  • Edit existing sessions
  • Photo display
Option C · Stats Focus

Richer Stats Tab

Dedicated Stats page with charts — like the iOS Stats tab.

  • Separate Stats view in sidebar
  • Grade pyramid chart
  • Climbs per month timeline
  • Sends by area breakdown
  • Route type breakdown
  • Year-on-year comparison
  • Requires Chart.js or similar
  • Training sessions (separate feature)

Implementation note
Chart.js vs vanilla: The iOS Stats tab uses Swift Charts (built-in). For the web, we can either use Chart.js (60KB, CDN) for polished charts, or hand-roll bar/pyramid charts in plain HTML/CSS (as shown above — simpler, zero deps). The plain CSS approach was already used for the bar chart concept on the Training page.
A Stats nav item could be added to the sidebar under the existing View section. Clicking it would replace the table/training list with a full-width stats panel — similar to how the iOS Stats tab works.
